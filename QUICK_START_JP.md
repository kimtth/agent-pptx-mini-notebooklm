# PPTX Slide Agent

PPTX Slide Agent を使ってプレゼンテーション資料を作成するための日本語ガイドです。

## 概要

PPTX Slide Agent は、以下の情報をもとに PowerPoint プレゼンテーションを作成できます。

- チャットで入力した指示
- アップロードしたファイル
- Web ページや URL

アプリは 3 つの主要エリアで構成されています。

- 左パネル: エージェントと会話するチャット
- 中央パネル: 生成されたスライドのプレビューとエクスポート
- 右パネル: スライド、参照データ、テーマ設定の管理

## 利用前の確認

- ローカルのスライドプレビューを使うなら Windows を利用します。
- プレビュー生成には Microsoft PowerPoint デスクトップ版が必要です。
- 有効なモデル接続情報を用意してください。

GitHub Copilot CLI を別途インストールする必要はありません。

## 初回設定

初めてアプリを開いたら、次の手順で設定します。

1. 右上の Settings ボタンをクリックします。
2. チームまたは管理者から案内された接続情報を入力します。
3. Save で保存します。

設定画面には次のような項目があります。

- `GITHUB_TOKEN` 必須
- `AZURE_OPENAI_ENDPOINT`
- `AZURE_OPENAI_API_KEY`
- `AZURE_TENANT_ID`
- `MODEL_NAME` 必須
- `REASONING_EFFORT`

多くの環境では、重要なのは別途 Copilot CLI を入れることではなく、有効なモデル接続情報を設定することです。

- GitHub ホストのモデルを使う場合は、Copilot 利用権のある `GITHUB_TOKEN` を入力します。
- Azure OpenAI を使う場合は、チームから案内された Azure のエンドポイントや認証情報を入力します。`AZURE_OPENAI_ENDPOINT` には `/openai/v1` を含む完全な URL を入力してください。

どの値を入力すべきかわからない場合は、アプリのセットアップ担当者に確認してください。

## 基本的な使い方

### 1. ワークスペースを選ぶ

上部バーのワークスペースメニューから、次の操作ができます。

- 新しいプロジェクトを開始する
- ワークスペースフォルダーを選択または変更する
- 作業内容を `.pptapp` プロジェクトとして保存する
- 既存の `.pptapp` プロジェクトを開く

ワークスペースフォルダーには、プレビュー画像、選択した画像、生成結果などが保存されます。

### 2. 参照データを追加する

右パネルの `Context` タブを開きます。

追加できるもの:

- ファイル
- URL

これらの参照データをもとに、エージェントがスライドの内容を組み立てます。

対応している主な形式:

- CSV
- DOCX
- MD
- TXT

後からファイルや URL の内容を更新した場合は、`Recreate source contents for files and URLs` を使って再読み込みしてください。

### 3. ストーリー構成を選ぶ

右パネルの `Slides` タブを開きます。

スライド生成前に、必要に応じて以下を選択します。

- ビジネスフレームワーク
- ブランドスタイル

ビジネスフレームワークは資料全体の論理構成を決めます。ブランドスタイルは見た目の方向性を決めます。

`Custom Template` のブランドスタイルを選ぶと、PPTX テンプレートを読み込んでテーマ色や背景を再利用できます。

### 4. 最初のドラフトを作る

左側のチャットパネルで、`Brainstorm` をクリックします。

アプリがスライド案を生成し、右側の `Slides` タブにドラフトのスライド構成が表示されます。

### 5. スライド案を確認・調整する

`Slides` タブでは、各スライドの以下を確認できます。

- タイトル
- キーメッセージ
- 箇条書き
- レイアウトのヒント
- アイコンのヒント
- 画像検索クエリ

さらに、次の操作もできます。

- スライド削除
- 付録への移動
- フレームワーク変更
- ブランドスタイル変更

スライド案を修正したい場合は、チャットで続けて指示を出してください。

### 6. スライド画像を追加する

各スライドには `Slide Image Search` セクションがあります。

入力できる内容:

- 単一のキーワード
- 1 行 1 キーワードの複数入力
- 画像の直接 URL
- ローカル画像ファイル

入力後、`Choose images` をクリックして、そのスライドに使う画像を 1 枚以上選択してください。

### 7. PowerPoint を生成する

スライド案の確認が終わったら、チャットパネルの `Create PPTX` をクリックします。

アプリは次の内容を使って資料を生成します。

- 承認済みのスライド構成
- テーマ設定とアイコン設定
- 選択済みのスライド画像
- PowerPoint 出力

生成が成功すると、プレビューも自動的に更新されます。

## テーマとカラーパレット

見た目をより細かく調整したい場合は、右パネルの `Palette` タブを使います。

できること:

- 最大 4 つのシードカラー追加
- パレット生成
- テーマスロット自動割り当て
- アイコンコレクション選択
- `.thmx` としてテーマをエクスポート

配色にこだわりたい場合は、PPTX 生成前にこのタブで調整してください。

## プレビューと出力

中央パネルには生成結果のプレビューが表示されます。

利用できる主な操作:

- `Refresh Preview`
- `.thmx`
- `Export .pptx`

プレビューを再読み込みしたい場合は `Refresh Preview` を使います。

完成した資料を保存したい場合は `Export .pptx` を使います。

## 作業内容を保存する

上部バーのワークスペースメニューから、現在の作業内容を `.pptapp` プロジェクトとして保存できます。

保存しておくと、次回以下を再開できます。

- スライド構成
- チャット履歴
- パレット設定

## トラブルシューティング

### エージェントが応答しない

Settings を開き、接続情報が正しく入力されているか確認してください。

この問題は、GitHub Copilot CLI の未インストールではなく、モデル接続情報の不足または誤りが原因であることがほとんどです。

### スライドプレビューが表示されない

Microsoft PowerPoint デスクトップ版がインストールされているか確認してください。表示されない場合は、`Refresh Preview` を試してください。

### PPTX 生成に失敗する

主な原因は次のいずれかです。

- 接続設定が不足している、または誤っている
- 必要なローカル依存関係が不足している
- 生成された資料でレイアウトまたは実行エラーが発生している

何度も失敗する場合は、ワークスペースの内容とあわせてアプリ管理者または保守担当者に連絡してください。

## 最初に試す流れ

はじめて使う場合は、次の流れがおすすめです。

1. アプリを開く
2. 必要な設定を入力する
3. ワークスペースフォルダーを選ぶ
4. `Context` で 1 つか 2 つのファイルまたは URL を追加する
5. `Slides` でフレームワークを選ぶ
6. チャットで資料作成を依頼し、`Brainstorm` をクリックする
7. スライド案を確認する
8. 必要なら画像を追加する
9. `Create PPTX` をクリックする
10. プレビューを確認し、最終的な `.pptx` をエクスポートする
